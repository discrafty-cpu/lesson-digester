Vinculum Fraction Rendering — Built Into Every Layout

All 10 layouts use FractionRenderer to convert inline text fractions to proper vertical notation. No cross-multiplication. No butterfly method. The vinculum (fraction bar) is always visible. The identity property is the universal engine.

3/4 → 34 | 5 (whole number) → 51 (invisible one) | 2/3 x 3/3 → 23 × 33 (giant one = 1)

Integration: FractionRenderer.convertTextToPptxRuns(text) for slides, .convertTextToHTML(text) for interactive pages, .drawPDFTextWithFractions(pdf, text, x, y) for PDF worksheets.

The Bookend Pattern — Slides That Appear Twice

Some layouts reappear at the start and end of a lesson, creating intentional symmetry. This is aligned with the Practice Profile's expectation that teachers "gather individual student progress using success criteria multiple times throughout the lesson."

Implementation Architecture
How these 10 layouts become code in the Lesson Digester

Existing: layout-engine.js

A centralized layout engine that replaces ad-hoc positioning. Each layout becomes a function that accepts content and theme, returning positioned PptxGenJS elements.

1
10 layout functions, each taking (slide, pres, content, C, theme)
2
Shared constants: MARGINS, GAPS, FONT_SCALE, HEADER_H, FOOTER_H
3
Layout registry: slideType → layoutFunction mapping

Existing: fraction-renderer.js

Vinculum fraction notation converter. All text with inline fractions passes through this before reaching any output target.

1
convertTextToPptxRuns(text) → superscript/subscript fraction runs for PPTX slides
2
convertTextToHTML(text) → vertical stacked fraction spans for interactive pages
3
drawPDFTextWithFractions(pdf, text, x, y) → stacked fractions in PDF worksheets
4
Helpers: invisibleOne(n), giantOne(n,d,f), complexFraction(a,b,c,d)

Updated: pptx-themes.js

Each theme extended with Practice Profile category colors, so layouts can color-code by domain automatically.

1
Add categoryColors: { unitDesign, community, clarity, delivery, assessment }
2
Each category gets a primary + tint color per theme
3
Layouts pull category color based on slide type

Build Order (Recommended)

Start with the layouts that cover the most slides and are most visible to students.
